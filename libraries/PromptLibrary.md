# Prompt Library

## Purpose

Prompt Library는 AI Builder OS에서 사용하는 모든 프롬프트를 관리하는 저장소이다.

프롬프트는 일회성 요청이 아니라 Builder OS의 핵심 자산이다.

모든 프롬프트는 버전 관리하고 지속적으로 개선한다.

---

## Principles

* 같은 프롬프트를 두 번 만들지 않는다.
* 프로젝트보다 재사용성을 우선한다.
* 프롬프트는 Workflow와 함께 개선한다.
* 검증된 프롬프트만 Library에 등록한다.

## Quality Checklist

등록 전 반드시 확인한다.

* 재사용 가능한가?
* 출력 형식이 일정한가?
* 다른 프로젝트에서도 사용할 수 있는가?
* Builder Workflow와 일치하는가?
* Playbook을 따르는가?

---

## Improvement Rules

프롬프트를 개선한 경우 반드시 아래를 수행한다.

* Version 증가
* 변경 이유 기록
* Prompt Library 업데이트
* Workflow 영향 검토

---

## Philosophy

좋은 프롬프트는 질문이 아니다.

좋은 프롬프트는 Builder OS의 실행 코드이다.

프롬프트를 많이 만드는 것이 목표가 아니라,

같은 프롬프트를 오래 사용하는 것이 목표다.
