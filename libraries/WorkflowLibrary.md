# Workflow Library

## Purpose

Workflow Library는 AI Builder OS에서 반복적으로 사용하는 업무 프로세스를 관리하는 저장소이다.

좋은 Workflow는 여러 프로젝트에서 재사용할 수 있어야 하며, 반복 작업은 반드시 Workflow로 전환한다.

---

## Principles

* 같은 작업을 두 번 하지 않는다.
* Workflow는 프로젝트보다 오래 살아야 한다.
* Workflow는 지속적으로 개선한다.
* 자동화 가능한 단계는 반드시 식별한다.

## Automation Strategy

Workflow마다 아래 항목을 기록한다.

Item	Description
Manual Steps	사람이 수행하는 작업
AI Steps	AI가 수행 가능한 작업
Automation	향후 자동화 가능한 작업
Current Status	Manual / AI Assisted / Automated

---

## Quality Checklist

Workflow 등록 전 확인한다.

* 다른 프로젝트에서도 사용할 수 있는가?
* 단계를 건너뛰지 않는가?
* Agent와 연결되는가?
* Prompt와 연결되는가?
* Playbook과 일치하는가?

## Philosophy

서비스는 일회성이다.

Workflow는 자산이다.

Builder OS의 경쟁력은 더 좋은 서비스를 만드는 것이 아니라,

더 좋은 Workflow를 축적하는 것이다.
